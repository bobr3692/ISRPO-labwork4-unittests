## 1. Цели и задачи тестирования

### Основные цели

- Автоматизация процесса проверки функциональности

### Задачи тестирования
- Разработать комплекс unit-тестов для функций `area()` и `perimeter()`
- Проверить обработку различных типов входных данных
- Протестировать граничные случаи и исключительные ситуации
- Обеспечить покрытие основных сценариев использования
- Верифицировать соответствие математическим расчетам

## 2. Описание тестируемого продукта

### Тестируемый модуль
**Файл:** `rectangle.py`  
**Тип:** Библиотека для вычисления площади и периметра прямоугольника

### Функциональность
- **Вычисление площади:** `S = a × b`
- **Вычисление периметра:** `P = 2 × (a + b)`
- **Поддержка типов данных:** неотрицательные целые числа, числа с плавающей точкой

## 3. Область тестирования

**Тестируемые функции:**
```python
def area(a, b):
    '''
    Возвращает значение площади прямоугольника
    Формула: S = a × b
    '''
    return a * b


def perimeter(a, b):
    '''
    Принимает длины сторон прямоугольника a и b
    Возвращает периметр: P = 2 × (a + b)
    '''
    return 2 * (a + b)
```

## 5. Критерии приемки

### Условия успешного завершения тестирования:
- Все разработанные unit-тесты выполняются без ошибок
- Покрытие кода тестами составляет 100% для тестируемых функций
- Отсутствуют критические и блокирующие дефекты
- Результаты вычислений соответствуют математическим ожиданиям
- Функции корректно обрабатывают все предусмотренные типы входных данных

### Критерии качества:
- Тесты независимы и могут выполняться в любом порядке
- Каждый тест проверяет одну конкретную функциональность
- Наличие понятных сообщений об ошибках
- Возможность повторного запуска тестов

### Критерии завершения:
- Выполнены все запланированные тестовые сценарии
- Подготовлен отчет о результатах тестирования
- Зафиксированы все обнаруженные дефекты
